# Contribution Guidelines

## 1. Branching Strategy
- **Main Branch:** Production ready code only. Protected.
- **Feature Branches:** Must be named `feat/ticket-number-description`.
- **Bugfix Branches:** Must be named `fix/ticket-number-description`.

## 2. Commit Message Standards
- Format: `[Type]: Short description (Ticket #)`
- Example: `[Feat]: Added login validation (Ref #42)`

## 3. Pull Request Process
- All PRs require 1 approval from a Code Owner.
- CI/CD tests must pass before merging.
